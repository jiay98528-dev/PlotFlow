# PlotFlow 首次用户旅程测试反馈

测试日期：2026-06-14  
测试目标：以第一次接触 PlotFlow 的用户视角，制作一个多分枝恋爱模拟互动小说文案内容并导出。  
测试入口：优先使用 Codex 内置浏览器；内置浏览器崩溃后，使用项目本地 Vite renderer (`http://127.0.0.1:5186`) 继续验证同一用户旅程。  
测试环境：Windows / PowerShell / Vite renderer / Playwright Chromium 辅助验证。

## 测试结论

当前用户旅程未能完整闭环。用户可以看到首页、打开“新建”弹窗、从模板创建故事、查看大纲和分支图，也能打开导出弹窗；但在 Codex 内置浏览器中页面直接崩溃，在普通浏览器预览模式下导出失败，无法完成“制作并导出”的核心任务。

## 关键问题

### P0：Codex 内置浏览器访问本地应用直接崩溃

复现步骤：
1. 启动本地 renderer：`vite packages/app --host 127.0.0.1 --port 5186 --strictPort`。
2. 使用 Codex 内置浏览器打开 `http://127.0.0.1:5186`。

实际结果：
- 内置浏览器 tab 导航超时。
- 页面变为 Chrome 崩溃页，标题为 `This page crashed`。
- 崩溃页显示 `blank crashed unexpectedly`。

期望结果：
- 首次用户能正常打开 PlotFlow 首页。

用户影响：
- 如果测试/演示依赖内置浏览器，用户旅程第一步即失败。
- 无法在内置浏览器中继续完成创建、编辑、导出操作。

证据：
- 浏览器状态：`This page crashed`
- 导航目标：`http://127.0.0.1:5186`

备注：
- 这可能与 Monaco、worker、CSP 或内置浏览器运行时兼容性有关，需要进一步定位。

### P0：浏览器预览模式下导出失败

复现步骤：
1. 打开 `http://127.0.0.1:5186`。
2. 点击“新建”。
3. 使用默认模板创建故事。
4. 点击顶部“导出”。
5. 保持默认 JSON 格式，点击导出弹窗底部“导出”按钮。

实际结果：
- 导出弹窗显示错误：`导出异常: Cannot read properties of undefined (reading 'file')`
- 页面未下载或保存任何导出文件。
- `window.plotflow` 在浏览器预览模式下不存在。

期望结果：
- 如果处于 Electron 环境：应打开系统保存对话框并保存 JSON/HTML/TXT。
- 如果处于浏览器预览环境：应禁用导出按钮、给出“请在桌面应用中导出”的明确提示，或降级为浏览器下载。

用户影响：
- “制作并导出”核心闭环失败。
- 首次用户不知道问题来自预览模式还是自己操作错误。

证据截图：
- `test-results/manual-export-dialog.png`
- `test-results/manual-export-after-click.png`

### P1：Monaco worker 初始化失败，可能导致编辑器卡顿或功能降级

复现步骤：
1. 打开 `http://127.0.0.1:5186`。
2. 等待 Monaco 编辑器加载。
3. 查看控制台日志。

实际结果：
控制台出现以下警告：
- `Could not create web worker(s). Falling back to loading web worker code in main thread, which might cause UI freezes.`
- `You must define a function MonacoEnvironment.getWorkerUrl or MonacoEnvironment.getWorker`

期望结果：
- Monaco worker 正常注册。
- 编辑器语法高亮、诊断、补全等功能不应退化到主线程。

用户影响：
- 长篇互动小说编辑时可能出现 UI 卡顿。
- 补全/诊断体验可能不稳定。

证据截图：
- `test-results/manual-journey-home.png`

### P1：CSP 阻止 Monaco 内联 SVG 资源加载

复现步骤：
1. 打开应用并创建模板故事。
2. 观察控制台日志。

实际结果：
控制台出现 CSP 错误：
- `Loading the image 'data:image/svg+xml,...' violates the following Content Security Policy directive: "default-src 'self'".`
- `img-src` 未显式配置，导致 Monaco 的 data SVG 被 `default-src` 拦截。

期望结果：
- CSP 应允许 Monaco 所需的安全资源，例如在 `img-src` 中允许 `data:`。
- 或使用非 data URI 的本地资源。

用户影响：
- 可能影响诊断波浪线、图标或编辑器视觉标记显示。
- 控制台持续报错会干扰开发和测试定位。

### P1：源码中存在明显中文乱码/转码痕迹

观察位置：
- `packages/app/src/renderer/App.tsx`
- `packages/app/src/components/panels/ExportDialog.tsx`
- `packages/app/src/components/editor/MonacoEditor.tsx`

实际结果：
- 多处注释和部分字符串在源码中呈现为 mojibake，例如 `鏂板缓`、`瀵煎嚭` 等。
- 当前 Vite 页面中部分用户可见中文是正常的，但源码可维护性已受影响。

期望结果：
- 源码文件统一以 UTF-8 保存。
- 所有中文注释、文案、测试断言均保持可读。

用户影响：
- 开发者难以维护 UI 文案和错误提示。
- E2E 测试中也存在乱码断言，降低测试可信度。

### P2：首次新建弹窗默认选中“RPG 对话”，不符合当前恋爱模拟创作目标

复现步骤：
1. 点击“新建”。
2. 查看模板弹窗默认预览。

实际结果：
- 默认预览为“RPG 对话”模板。
- 视觉小说模板需要用户主动选择。

期望结果：
- 如果目标用户常见场景包含恋爱模拟/视觉小说，可以考虑让“视觉小说”更显眼，或在模板卡片上提供类型筛选。

用户影响：
- 首次用户需要理解模板差异后再选择。
- 对“恋爱模拟互动小说”场景，默认模板路径不够直接。

证据截图：
- `test-results/manual-journey-02-new-dialog.png`

### P2：浏览器预览模式状态提示不够指导用户

复现步骤：
1. 在浏览器预览模式打开应用。
2. 查看底部状态栏。

实际结果：
- 底部显示 `Browser preview mode`。
- 但导出按钮仍可点击，并在失败后显示底层异常。

期望结果：
- 浏览器预览模式应明确说明哪些能力不可用。
- 对不可用功能应禁用入口或给出可操作替代方案。

用户影响：
- 首次用户会把环境限制误认为产品功能损坏。

## 成功路径观察

以下功能在普通浏览器 renderer 模式下可用：
- 首页正常显示，中文 UI 可读。
- “新建”弹窗可打开。
- 模板列表和预览可见。
- 默认模板创建后，大纲与分支图能同步显示节点。
- 导出弹窗可打开，JSON/HTML/TXT 三种格式可见。

## 测试限制

- Codex 内置浏览器在打开本地应用时直接崩溃，因此无法在该浏览器中完成完整点击式旅程。
- 后续操作使用 Playwright Chromium 对同一 Vite renderer 进行验证。
- PowerShell 到 Node 的管道对脚本内中文字符串存在转码问题；因此“手动注入中文恋爱模拟文本”的自动化结果中出现 `??`，该现象不直接归类为 PlotFlow 产品 BUG。模板内置中文显示正常。

## 建议修复优先级

1. 先定位内置浏览器崩溃原因，重点检查 Monaco worker、CSP、内置浏览器对本地 Vite/worker 的限制。
2. 导出逻辑增加环境保护：`window.plotflow?.file?.saveExport` 不存在时禁用导出或使用浏览器下载 fallback。
3. 修复 Monaco worker 配置，显式设置 `MonacoEnvironment.getWorker` 或按 Vite 官方方式拆分 worker。
4. 调整 CSP，至少为 Monaco 所需图像资源配置 `img-src 'self' data:`。
5. 全仓排查并修复 UTF-8 乱码源码和 E2E 乱码断言。
6. 浏览器预览模式下增加功能可用性提示，避免用户点击必然失败的 Electron-only 功能。
