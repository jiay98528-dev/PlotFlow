## 1. 产品概述

### 1.1 产品定位

Fablevia（维叙）是一个面向独立游戏开发者的**叙事分支管理工具**，以 `.mdstory` 纯文本文件为唯一磁盘真相源，以 Graph Lab 作为主要且默认的图优先创作工作区，并在顶栏并列保留 Split 完整源码投影，以多格式导出为目标。产品的核心价值是成为**文案策划、叙事设计师、程序开发三者之间的工作流桥梁**。

### 1.2 核心差异化

| 维度 | Fablevia（维叙） | 竞品 |
|------|----------|------|
| 学习成本 | Markdown 方言，零代码 | Ink 需学脚本语言，Articy 需培训 |
| 可视化 | Graph Lab 默认图优先工作区 + Split 实时分支图源码投影 | Twine 可视化但不可编辑连线 |
| 程序集成 | 干净 JSON 导出 + 引擎插件 | Twine 格式封闭，Ink 仅 Unity |
| 隐私 | 离线优先，本地运行 | 部分竞品需联网 |
| 价格 | $29 买断 | Articy €499/年起 |
| AI 辅助 | 概率补全（统计模型，不联网） | 无同类功能 |

### 1.3 目标用户

- **首要用户**：独立游戏开发者（单人/小型团队，1-5人）
- **次要用户**：游戏设计院校师生、视觉小说/互动小说制作者
- **潜在用户**：桌游剧本作者（扩展场景）

### 1.4 产品形态

- **桌面应用**：Windows / macOS / Linux（Electron）
- **引擎插件**：Godot（V0.1 完整实现）、Unity（V0.1 接口+示例）、Unreal（V0.1 接口定义）
- **文件格式**：`.mdstory`（纯文本 Markdown 方言，Git 友好）

## 2. 用户画像与使用场景

### 2.1 核心用户画像

| 角色 | 技能水平 | 核心需求 | 使用模式 |
|------|---------|---------|---------|
| **文案策划** | 熟悉 Markdown，不会编程 | 自由写作，看到分支结构 | 独立模式：打开 .mdstory 就写 |
| **叙事设计师** | 理解分支逻辑，可能略懂脚本 | 可视化整体结构，快速定位节点 | 右侧分支图为主，编辑器为辅 |
| **程序开发** | 精通编程，不写文案 | 获取结构化数据导入引擎 | 插件模式：在引擎中打开，一键导出 |
| **团队协作者** | 混合技能 | 同一份文件，各自用自己的视图 | Git 管理 .mdstory，各看各的 |

### 2.2 典型使用场景

**场景 1：独立开发者单人创作**
> 小明在开发一款像素风 RPG。他打开 Fablevia（维叙），在 VSCode 风格的编辑器中用 Markdown 写剧情分支。写完一个章节后，右侧分支图自动显示所有节点关系。他发现"狼穴"节点没有出口选项（死胡同警告），于是补了一个逃跑选项。写完导出 JSON，丢进 Godot 项目，对话系统直接跑起来。

**场景 2：文案+程序双人协作**
> 文案小红在独立模式下写 `.mdstory`，用幽灵补全加速写作。写完推送到 Git。程序员小刚在 Godot 中通过 Fablevia 插件打开同一个文件，发现小红引用了一个未在引擎中定义的变量 `$好感度`，编辑器标红提示。他告诉小红加了变量定义，小红那边的补全自动开始推荐 `$好感度`。

**场景 3：Game Jam 快速原型**
> 48小时 Game Jam，3人小队。一人用 Fablevia（维叙）写叙事脚本（单文件模式），用了内置的"视觉小说模板"。写完直接导出 HTML 可玩版，队友在浏览器里点一遍验证所有分支。确认无误后导出 JSON，拖进 Unity 项目，15分钟完成对话系统接入。

### 3.2 详细功能规格

#### F3.1.1 Markdown 编辑器

| 属性 | 规格 |
|------|------|
| **基础能力** | 支持标准 Markdown 语法（标题、加粗、斜体、列表、分隔线） |
| **Fablevia 语法高亮** | `# 节点：` 蓝色标记，`[选项]` 绿色标记，`[条件]` 橙色标记，`$变量` 紫色标记 |
| **括号自动闭合** | `[选项]` 输入 `[` 后自动提示补全 |
| **节点折叠** | 可折叠/展开单个节点块（方便长文档导航） |
| **大纲视图** | 左侧边栏显示节点树（基于标题层级），点击跳转 |

#### F3.1.2 分支图实时渲染

| 属性 | 规格 |
|------|------|
| **渲染引擎** | React Flow（MIT协议，专为可编辑节点图设计） |
| **布局算法** | 自上而下树状布局（Dagre），根节点在顶部 |
| **节点卡片** | 显示节点标题+前30字摘要+选项数量徽章 |
| **连线样式** | 贝塞尔曲线，条件连线虚线，无条件连线实线 |
| **性能目标** | 200节点/1000选项规模下 ≥30fps |
| **超限处理** | 超过200节点时启用虚拟滚动+节点折叠 |
| **缩放** | 鼠标滚轮缩放（10%~200%），Ctrl+0 恢复默认 |

#### F3.1.3 图形化条件编辑器

| 属性 | 规格 |
|------|------|
| **触发方式** | 点击选项行右侧的 `[🔧条件]` 图标 |
| **面板形态** | 内联弹出面板（不遮挡编辑器主体） |
| **交互模式** | Airtable 过滤条件构建器风格 |
| **组件** | 变量下拉框（读取 Frontmatter 声明或引擎变量） + 比较运算符下拉（==, !=, >, <, >=, <=）+ 值输入框 |
| **逻辑组合** | AND/OR 按钮，支持添加条件组（最多嵌套3层） |
| **双向同步** | 面板修改 → 自动更新编辑器中 `条件: (...)` 文本行；手动编辑文本 → 面板刷新 |

#### F3.1.4 选项辅助工具

| 属性 | 规格 |
|------|------|
| **熟练用户** | 直接手写 `[选项] 描述 -> 节点：目标` 语法 |
| **新手用户** | 工具栏按钮「+选项」或右键菜单「插入选项」→ 弹出迷你表单：选项描述 + 跳转目标下拉（自动列出所有节点）→ 插入格式化文本 |
| **补全辅助** | 输入 `[选项]` 后，幽灵字符自动建议常见句式 |

#### F3.1.5 实时语法错误标记

| 属性 | 规格 |
|------|------|
| **错误标记** | 红色波浪线（阻断性错误）+ 左侧边栏红色标记点 |
| **警告标记** | 黄色波浪线 + 左侧黄色标记点 |
| **建议标记** | 蓝色下划线 + 左侧蓝色标记点 |
| **悬停提示** | 鼠标悬停在标记上显示错误说明 tooltip |
| **分支图联动** | 孤立节点在分支图中黄色高亮，死胡同灰色高亮 |

#### F3.1.6 Graph Lab（图优先正式入口）

| 属性 | 规格 |
|------|------|
| **定位** | 主要且默认的流程图优先工作区，支持“完全 GUI 操控”覆盖核心写作闭环 |
| **入口** | 首次启动、新建、打开、命令行打开和 Continue editing 默认进入；顶部与 Split 并列的模式按钮及快捷键允许随时切换 |
| **画布** | React Flow 全屏画布，左侧节点 palette，右侧 Inspector，底部 ProblemPanel 联动 |
| **节点编辑** | GUI 创建/删除节点；Inspector 编辑章节、标题、正文摘要；拖拽节点实时移动，松手写入 `.mdstory` 布局块 |
| **选项编辑** | GUI 创建/删除/排序选项；拖线连接已有节点、拖到空白处创建目标节点、拖到空白处断开既有连接 |
| **流程出口** | 无选项节点显示默认节点级连线口，拖线写入 `下一步: 节点：目标`；一旦存在普通选项，默认连线口隐藏 |
| **变量编辑** | `vars:` 是当前单文件全局变量源；Graph Lab Inspector 必须能查看、新增、删除变量，并在条件/效果编辑器中以下拉方式引用 |
| **条件/效果** | 条件编辑器嵌入 Inspector，效果编辑器以同样的字段化方式呈现；条件和效果均从已声明变量列表中选择变量并写回源码 |
| **章节工作区** | 顶部必须显示章节标签栏；每个章节拥有独立图表上下文和 Source Drawer 源码切片，新增章节后立即出现可见标签 |
| **源文本** | Source Drawer 可折叠显示当前章节 `.mdstory` 源文本切片；基础文本编辑模式保留全文件视图；所有 GUI 操作必须序列化回同一个 `.mdstory` |
| **可见性门禁** | Graph Lab E2E 必须用截图验证章节标签栏和新增章节标签可见，避免仅通过 DOM 存在误判 UI 已呈现 |
| **工作区关系** | Graph Lab 是主要且默认工作区；Split 顶栏并列保留，作为辅助与高级的完整 `.mdstory` 源码投影 |

#### F3.2.1 四维幽灵字符补全

| 维度 | 触发条件 | 补全内容 | 示例 |
|------|---------|---------|------|
| **节点标题** | 输入 `# 节点：` 后 | 基于语料库推荐节点名称 | `森林入` → `森林入口/森林深处/森林边缘` |
| **选项句式** | 输入 `[选项]` 后 | 基于语料库推荐选项描述 | `走向左` → `走向左边的狼嚎声/走向左侧的小路` |
| **正文描述** | 正文写作中随时 | 根据上下文补全环境/动作描写 | `你站在` → `你站在幽暗森林的边缘，` |
| **变量名** | 输入 `$` 后 | 读取 Frontmatter 或引擎变量列表 | `$好` → `$好感度` |

| 属性 | 规格 |
|------|------|
| **展示方式** | 灰色幽灵字符（ghost text），与 FlettEN AI 模式一致 |
| **接受** | Tab 键接受当前建议 |
| **忽略** | Esc 键忽略，继续输入自动消失 |
| **多候选** | Ctrl+Space 打开下拉菜单，方向键选择，Enter 确认 |
| **频率控制** | 输入间隔 < 100ms 不触发（避免快速输入时闪烁） |

## 4. 语法规范

### 4.1 YAML Frontmatter（变量声明）

每个 `.mdstory` 文件以 YAML Frontmatter 开头，声明所有变量和类型：

```yaml
---
plotflow: "0.1"
title: "暗夜森林"
author: "文案名"
engine: "godot"           # 可选: generic | godot | unity | unreal
layout:                   # 可选：Graph Lab / 分支图手动布局
  graph:
    version: 1
    nodes:
      - id: "%E7%AC%AC%E4%B8%80%E7%AB%A0/%E6%A3%AE%E6%9E%97%E5%85%A5%E5%8F%A3"
        x: 120
        y: 80
vars:
  好感度: int
  金币: int
  职业: enum[战士, 法师, 盗贼]
  警戒值:
    type: int
    default: 0
    scope: chapter
    chapter: 第一章
  装备: object{武器: enum[剑, 弓, 杖], 护甲: int, 饰品: object{名称: string, 效果: string}}
  钥匙: bool
  日志: string
---
```

#### Graph Layout 块（可选）

`layout.graph.nodes` 是 `.mdstory` Frontmatter 中的可选布局投影，用于保存 Graph Lab 或分支图中的手动节点坐标。它不改变剧情语义，不影响运行时分支逻辑；缺失时编辑器继续用 Dagre 自动布局。

| 字段 | 规格 |
|------|------|
| `layout.graph.version` | 当前固定为 `1`，用于未来布局格式迁移 |
| `layout.graph.nodes[].id` | canonical opaque FullID：`encodeURIComponent(chapterId) + "/" + encodeURIComponent(nodeId)` |
| `layout.graph.nodes[].x/y` | React Flow 画布坐标，允许整数或浮点数 |
| 写入时机 | 用户拖拽节点松手、GUI 创建节点并带落点、节点重命名/删除时同步迁移或清理 |

`layout.graph.version` 继续为 `1`。读取历史 `chapter-node` ID 时不得拆分字符串猜测；必须枚举当前 AST 节点计算 legacy aliases，仅唯一命中时迁移坐标，碰撞时忽略该坐标并报告歧义。打开文件不静默改写，下一次真实保存或布局事务才写回 canonical ID。

#### 类型系统规格

| 类型 | 语法 | 默认值 | 嵌套 |
|------|------|--------|------|
| `int` | `变量名: int` | 0 | - |
| `float` | `变量名: float` | 0.0 | - |
| `bool` | `变量名: bool` | false | - |
| `string` | `变量名: string` | "" | - |
| `enum` | `变量名: enum[值1, 值2, ...]` | 第一个值 | - |
| `object` | `变量名: object{字段: 类型, ...}` | 各字段默认值 | 最多3层 |

#### 嵌套规则

- `object` 的字段类型可以是 `int`, `float`, `bool`, `string`, `enum` 或另一个 `object`
- 最大嵌套深度：**3层**（外层 object → 内层 object → 最内层 object）
- 超过3层的嵌套在解析时报错
- `scope` / `chapter` 只允许在顶层变量声明；`scope: chapter` 必须选择真实 `chapter`，global 或省略 scope 时禁止 `chapter`
- object fields 继承顶层变量作用域；chapter-scoped 值随故事会话持久化，但只在归属章节可见

### 4.2 章节与节点定义

```markdown
# 第一章：村庄          ← H1 = 章节标题
## 节点：森林入口        ← H2 = 节点定义
### 选项区               ← H3 = 选项区标记（可选，解析器忽略）

你站在幽暗森林的边缘，两条小径延伸向前。
空气潮湿，远处传来低沉的狼嚎。

[选项] 走向左边的狼嚎声 -> 节点：狼穴
  条件: ($开锁技能>=5) AND ($有工具==true)
  效果: (勇气+1)

[选项] 探索右边的古井 -> 节点：古井
  （无条件=默认选项）

[选项] 返回村庄 -> 节点：村庄广场
  效果: (好感度-1)
```

#### 节点规则（强制）

| 规则编号 | 规则内容 | 违反处理 |
|----------|---------|---------|
| N1 | 节点ID必须全局唯一。跨章节不得重名 | 🔴 错误 |
| N2 | 节点引用格式：同章节内可直接用 `节点：名称`，跨章节必须用 `章节名/节点：名称` | 🔴 错误（找不到目标） |
| N3 | 章节标题格式 `# 章节名`，节点标题格式 `## 节点：节点名` | 🟡 警告（格式不规范） |
| N4 | 每个节点必须有至少一个无条件的选项，或所有条件至少有一个可达路径 | 🔵 建议（可能卡关） |
| N5 | 节点描述文本（正文）不能为空 | 🟡 警告 |

### 4.3 选项语法

```
[选项] <描述文本> -> 节点：<目标节点名>
  条件: <逻辑表达式>
  效果: (<变量操作列表>)
```

#### 选项规则

| 规则编号 | 规则内容 | 违反处理 |
|----------|---------|---------|
| O1 | `-> 节点：目标` 必须有，且目标节点必须存在 | 🔴 错误 |
| O2 | 条件子行以 `条件:` 开头，缩进（2空格或1Tab） | 🔴 错误（解析失败） |
| O3 | 效果子行以 `效果:` 开头，缩进（2空格或1Tab） | 🔴 错误（解析失败） |
| O4 | 一个选项可以同时有条件和效果，也可以只有其一 | - |
| O5 | 无条件子行的选项 = 默认选项，始终可用 | - |
| O6 | 同一节点下所有选项的描述文本不能完全相同 | 🟡 警告 |

### 4.3.1 流程出口语法（`下一步`）

`下一步` 是节点级默认流程出口，用于“没有显式选项也要进入下一个节点”的流程性节点。它不是普通 `[选项]` 的伪装，必须作为节点属性解析、验证和写回。

```markdown
## 节点：开场旁白

远处传来钟声。

下一步: 节点：村口
  效果: (时间+1)
```

| 规则编号 | 规则内容 | 违反处理 |
|----------|---------|---------|
| F1 | `下一步:` / `下一步：` 后必须是合法节点引用，格式同普通选项目标 | 🔴 错误 |
| F2 | 紧邻缩进的 `效果:` 行归属于该流程出口 | - |
| F3 | 一个节点存在任意 `[选项]` 时，Graph Lab 关闭默认流程连线口，以普通选项作为显式出口 | - |
| F4 | JSON 导出在 schema 升级前将 `下一步` 投影为文本为 `下一步` 的无条件合成 option，保持运行时兼容 | - |

### 4.4 变量操作语法（效果）

在 `效果:` 后的括号内，支持三种操作：

| 操作 | 语法 | 示例 | 适用类型 |
|------|------|------|---------|
| 赋值 | `变量名=值` | `武器='长剑'` | 所有类型 |
| 增减 | `变量名+值` / `变量名-值` | `好感度+3`, `金币-10` | int, float |
| 追加 | `变量名←值` | `日志←'获得了钥匙'` | string（追加到末尾） |

多个操作用逗号分隔：`(好感度+3, 金币-10, 武器='长剑', 钥匙=true)`

### 4.5 条件表达式语法

#### 文本表示（存储在 .mdstory 中）

```
条件: ($变量 运算符 值) AND ($变量 运算符 值)
条件: ($变量 运算符 值) OR ($变量 运算符 值)
条件: NOT ($变量 运算符 值)
```

| 元素 | 语法 | 示例 |
|------|------|------|
| 比较运算符 | `==`, `!=`, `>`, `<`, `>=`, `<=` | `$好感度 >= 5` |
| 逻辑与 | `AND` | `($a==true) AND ($b>=3)` |
| 逻辑或 | `OR` | `($a==true) OR ($b>=3)` |
| 逻辑非 | `NOT` | `NOT ($a==true)` |
| 括号分组 | `()` | `(($a AND $b) OR $c)` |
| 字段访问 | `.` | `$角色.职业 == '战士'` |

#### 图形化编辑（用户实际交互）

用户不手写条件表达式，而是通过弹出面板构建。面板组件到文本的双向同步由编辑器自动处理。

|------|------|---------|
| **正常** | 绿色边框 `#4EC9B0` | 有入口有出口，无条件或条件可达 |
| **孤立** | 黄色边框 `#DCDCAA` | 没有任何选项指向该节点（除了根节点） |
| **死胡同** | 灰色边框 `#808080` | 节点没有出口选项 |
| **错误** | 红色边框 `#F44747` | 节点包含语法错误或引用不存在目标 |
| **当前选中** | 蓝色光晕 `#569CD6` | 编辑器光标正在该节点内 |

### 6.4 交互行为

| 操作 | 行为 |
|------|------|
| **单击节点** | 编辑器自动滚动到对应 `## 节点：` 位置，节点高亮 |
| **双击节点** | 编辑器滚动 + 进入重命名模式（修改节点标题） |
| **拖拽连线端点** | 将连线从当前目标拖到另一个节点 = 修改选项的跳转目标，文本自动更新 |
| **拖线到空白处** | 打开动作菜单：创建普通节点并连接、创建结局节点并连接、搜索已有节点并连接、取消 |
| **拖既有线缆到空白处** | 打开动作菜单：断开此选项、创建节点并重连、搜索已有节点并重连、取消 |
| **右键节点** | 弹出菜单：跳转到节点 / 重命名 / 添加选项 / 删除节点 |
| **右键空白** | 弹出菜单：添加新节点 / 重新布局 / 切换到 Graph Lab |
| **Ctrl+点击节点** | 多选节点（用于批量操作） |
| **滚轮** | 缩放 |
| **中键拖拽** | 平移画布 |

### 6.5 布局规则

1. **根节点**（第一个 `## 节点：`）固定在画布顶部中央
2. **同层节点**水平等距分布（最小间距 150px）
3. **父子关系**垂直间距 120px
4. **连线交叉最小化**：使用 Dagre 的层级排序算法
5. **孤立节点**放置在画布右侧独立区域
6. 超过 20 个同层节点时启用水平折叠

### 8.3 JSON Schema 字段说明

| 路径 | 类型 | 必须 | 说明 |
|------|------|------|------|
| `meta.plotflow` | string | ✅ | 格式版本号（保留的 `plotflow` 兼容字段） |
| `meta.title` | string | ✅ | 故事标题 |
| `meta.author` | string | ❌ | 作者名 |
| `meta.engine` | string | ✅ | JSON 输出目标：`godot`/`unity`/`unreal`/`none`；源/内部 `generic` 映射为 `none` |
| `variables.<name>.type` | string | ✅ | `int`/`float`/`bool`/`string`/`enum`/`object` |
| `variables.<name>.scope` | string | ❌ | 仅顶层：`global`/`chapter`；省略视为 global |
| `variables.<name>.chapter` | string | 条件必需 | 仅 `scope=chapter` 时必需，必须引用真实章节 |
| `variables.<name>.fields` | object | object 必需 | object 始终输出；无字段时也必须是 `{}`，嵌套字段继承顶层 scope/chapter |
| `chapters[].nodes[].id` | string | ✅ | 节点 ID（不含章节前缀） |
| `chapters[].nodes[].fullId` | string | ✅ | encoded-slash canonical opaque FullID；禁止自行拆分 |
| `chapters[].nodes[].body` | string[] | ✅ | 描述文本数组（按段落） |
| `chapters[].nodes[].options[].targetNodeId` | string\|null | ✅ | 无目标时为 null |
| `chapters[].nodes[].options[].targetChapterId` | string\|null | ✅ | 显式章节；同章未限定时为 null |
| `chapters[].nodes[].options[].targetFullId` | string\|null | ✅ | canonical opaque FullID；无目标时为 null |
| `chapters[].nodes[].options[].conditions` | object\|null | ❌ | 条件表达式（含 AST） |
| `chapters[].nodes[].options[].conditions.ast` Comparison | object | 条件存在时 | 0.2 使用 `left` / `right` typed operands：`variable{name}` 或 `literal{value}` |
| `chapters[].nodes[].options[].sideEffects` | array | ❌ | 变量副作用列表 |
| `chapters[].nodes[].options[].sideEffects[].operation` | string | ✅ | `set`/`add`/`subtract`/`append` |
| `chapters[].nodes[].isRoot` | bool | ✅ | 是否为故事起始节点 |
| `chapters[].nodes[].isOrphan` | bool | ✅ | 是否为孤立节点（诊断信息） |
| `chapters[].nodes[].isDeadEnd` | bool | ✅ | 是否为死胡同（诊断信息） |

## 9. 错误检测系统

### 9.1 三级错误分类

#### 🔴 错误（Error）— 阻断导出

| 编号 | 错误类型 | 检测条件 | 标记位置 |
|------|---------|---------|---------|
| E001 | 未定义目标节点 | 选项指向的节点ID不在任何章节中 | `-> 节点：XXX` 处 |
| E002 | 未声明变量 | 条件/效果中引用了未在 Frontmatter 中声明的变量 | `$变量` 处 |
| E003 | 枚举值非法 | 赋值的枚举值不在 Frontmatter 声明的合法值列表中 | 效果括号内 |
| E004 | 类型不匹配 | 效果中操作的值类型与变量声明类型不一致 | 效果括号内 |
| E005 | 语法解析失败 | 行无法被解析器识别（如缺少 `->`） | 整行 |
| E006 | 嵌套深度超限 | object 嵌套超过3层 | Frontmatter 中 |
| E007 | 节点ID重名 | 两个节点使用相同的全局ID | 重复的 `## 节点：` 行 |
| E008 | 变量重复声明 | Frontmatter 中同一变量声明两次 | Frontmatter 中 |

#### 🟡 警告（Warning）— 不阻断但需关注

| 编号 | 警告类型 | 检测条件 | 标记位置 |
|------|---------|---------|---------|
| W001 | 孤立节点 | 节点不是根节点，且没有任何选项指向它 | 节点标题处 |
| W002 | 死胡同节点 | 节点没有任何出口选项，且不是最终结局 | 节点标题处 |
| W003 | 未使用变量 | Frontmatter 声明的变量在全文未被引用 | Frontmatter 中 |
| W004 | 重复选项描述 | 同一节点下两个选项的文本完全相同 | 选项行 |
| W005 | 空描述节点 | 节点 body 文本为空或仅空白字符 | 节点标题处 |
| W006 | 格式不规范 | 章节/节点标题格式不符合约定 | 标题行 |
| W007 | 可能无限循环 | 选项边和 `下一步` 边形成无外部出口的强连通闭环 | 闭环节点标题处 |

#### 🔵 建议（Info）— 最佳实践提醒

| 编号 | 建议类型 | 检测条件 | 标记位置 |
|------|---------|---------|---------|
| I001 | 可能卡关 | 节点的所有选项都有条件，无兜底选项 | 节点标题处 |
| I002 | 描述过短 | 节点 body 文本少于10个字符 | 节点标题处 |
| I003 | 无章节归属 | 根节点之前没有章节标题 | 文件开头 |

### 9.2 呈现方式

| 元素 | 错误 | 警告 | 建议 |

## 10. 技术架构

### 10.1 技术栈选型

| 层级 | 技术 | 选型理由 |
|------|------|---------|
| **桌面框架** | Electron 42+ | 当前受支持稳定主版本线，复用现有编辑器资产，与 React Flow 完全兼容 |
| **前端框架** | React 18+ + TypeScript 5+ | React Flow 原生 React 组件，TypeScript 保证类型安全 |
| **状态管理** | Zustand | 轻量，适合编辑器场景的细粒度状态 |
| **UI 组件库** | 自研组件 + Radix UI（无障碍） | 保持编辑器个性，Radix 提供菜单/对话框等无样式原语 |
| **代码编辑器** | Monaco Editor（VSCode 内核） | 语法高亮、波浪线标记、幽灵补全均原生支持 |
| **分支图** | React Flow + Dagre | 可编辑节点图 + 层级布局算法 |
| **Markdown 解析** | unified + remark + 自定义插件 | 成熟的 Markdown AST 生态，自定义语法通过插件实现 |
| **YAML 解析** | js-yaml | 解析 Frontmatter，成熟稳定 |
| **本地存储** | better-sqlite3（语料索引/学习模型） | 嵌入式关系型，无需安装 |
| **打包** | electron-builder | Windows/macOS/Linux 三平台 |
| **测试** | Vitest（单元）+ Playwright（E2E） | 与 Vite/Electron 生态兼容 |

### 10.2 为什么是 Electron 而不是 Tauri

| 考量 | Electron | Tauri |
|------|----------|-------|
| 现有资产复用 | ✅ 策划文档明确提及"复用现有 Electron 编辑器" | ❌ 需完全重写 |
| Monaco Editor | ✅ 官方支持 | ⚠️ 需额外适配（WebView 兼容性） |
| React Flow | ✅ 浏览器原生交互 | ✅ Tauri 内置 WebView 支持 |
| Godot/Unity 集成 | 无影响 — 引擎集成通过 JSON 插件，与编辑器框架无关 | 同左 |
| 开发效率 | 12人天可完成 | 增加 5-7 人天（学习+Tauri API 适配） |
| 包体积 | ~120MB（工具类软件不敏感） | ~10MB |
| 结论 | **V0.1 选择 Electron**，Tauri 作为 V0.3 迁移选项 | |

## 11. 项目结构与文件格式

### 11.1 .mdstory 文件格式

`.mdstory` 是 UTF-8 编码的纯文本文件，MIME 类型为 `text/markdown; variant=plotflow`。

**文件结构**：
```

变量声明
```

**扩展名注册**：
- Windows: `.mdstory` 关联到 Fablevia.exe
- macOS: `.mdstory` 关联到 Fablevia.app
