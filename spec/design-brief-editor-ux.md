# PlotFlow 编辑器 UX 设计简报

> **类型**：`/impeccable shape` 产出
> **日期**：2026-06-12
> **版本**：V1.0
> **状态**：待确认
> **范围**：PlotFlow 编辑器全部页面和模块（M0-M7）

---

## 1. Feature Summary

PlotFlow 是一个面向独立游戏开发者的叙事分支管理工具。编辑器以 Markdown 方言（`.mdstory`）为核心，左侧大纲导航，中央 Monaco 编辑器，右侧 React Flow 分支图三栏布局。核心闭环：写 Markdown → 实时分支图渲染 → 图形化条件编辑 → 语法错误检测 → 幽灵补全 → 多格式导出。

**优先级用户**：叙事设计师——需要同时看到文本结构和分支拓扑的人。文案策划和程序开发各自在其面板中获得最佳体验。

**核心张力**：功能很多（6 大模块 × 若干子面板），但第一印象必须轻盈。解决策略：渐进式披露 + 游戏化入门 + 语义多色编码让信息有层次而非堆砌。

---

## 2. Primary User Action

叙事设计师打开一个 `.mdstory` 文件。编辑器中看到语法高亮的剧情文本。右侧分支图实时展示节点结构，迷你地图模式不抢注意力。他点击一个节点卡片——编辑器跳转到对应行。他拖拽连线端点修改跳转目标——文本 `-> 节点：XXX` 自动更新。他点击选项行右侧的 `[🔧条件]` 图标——条件编辑器以弹出面板呈现，构建一个 AND 条件组后点击应用——编辑器中的 `条件:` 子行自动刷新。半小时完成章节的分支逻辑调整，Ctrl+E 导出 JSON，丢给程序员。

## 3. Design Direction

### 3.1 色彩策略

**语义多色（Information-density）**

| 层级 | 色值逻辑 | 用途 |
|------|---------|------|
| 底色 | 中性灰 `#F5F5F6`（亮色）/ `#1A1B1E`（暗色） | 编辑器背景，不冷不暖 |
| 表面 | 纯白 `#FEFEFE`（亮色）/ `#212327`（暗色） | 面板、卡片、弹出层 |
| 文字 | `#1B1C1E` / `#E1E2E4` | 主文字 |
| 次要文字 | `#6B6D72` / `#9B9DA2` | 描述、标签 |
| **语义色** | | |
| 节点标题 | `#2563EB` (蓝) | `# 节点：` / `## 节点：` 标记 |
| 选项 | `#16A34A` (绿) | `[选项]` 标记 |
| 条件 | `#D97706` (琥珀) | `条件:` 子行、条件连线 |
| 效果 | `#CA8A04` (金) | `效果:` 子行 |
| 变量 | `#9333EA` (紫) | `$变量名` 引用 |
| 跳转 | `#0891B2` (青) | `-> 节点：` 目标引用 |
| 错误 | `#DC2626` (红) | 阻断性错误波浪线 |
| 警告 | `#F59E0B` (琥珀) | 警告波浪线 |
| 建议 | `#6366F1` (靛) | 建议下划线 |
| **强调色** | `#2563EB` (蓝) | CTA 按钮、选中态、链接 |

**设计原理**：每种语法元素有独立的色彩身份——用户不读文字也能通过颜色判断这一行是什么。Figma 图层面板的色彩哲学：颜色减认知负荷，不加它。

### 3.2 主题

**默认亮色**，暗色作为可切换选项（Ctrl+Shift+T）。

**场景句**：
> 叙事设计师在下午三点的工作室，27 寸显示器前，自然光从窗户斜射进来，手边一杯已经凉了的咖啡。他需要在两小时内理清第三章 20 个节点的分支逻辑。工具应该像一盏台灯——照亮他的文字，不抢夺注意力。

### 3.3 命名锚点

| 锚点 | 借鉴什么 |
|------|---------|
| **Figma** | 图层面板的色彩语义编码——每种节点类型一个颜色，不看标签也能导航 |
| **Notion（亮色模式）** | 长文本编辑的舒适度——行高、字号、留白策略。写作工具不是 IDE |
| **Observable** | 代码+可视化并排的信息密度管理——数据多但不乱，靠的是清晰的视觉层级 |

### 3.4 组件库

**shadcn/ui** — 基于 Radix 原语 + 预样式组件。复制源码到项目中，完全可定制。优势：
- 无障碍性由 Radix 保证（Dialog/Popover/Tooltip/Dropdown 的 ARIA 模式）
- 样式可以直接修改 Tailwind 类，适配语义多色策略
- 社区活跃（60k+ GitHub stars），Vercel 维护

### 3.5 反目标

> **绝不学 Articy:Draft** — 不做密密麻麻的属性面板、不做"配置前置税"、不做需要用户花数天设置模板才能开始工作的系统。

具体禁令：
- ❌ 单次操作需要 3 次以上点击
- ❌ 任何在首次启动时可见但普通用户 5 分钟内用不到的功能
- ❌ 固定的多行工具栏——最多一行
- ❌ 属性面板占据 30%+ 屏幕空间

## 4. Scope

| 维度 | 决定 |
|------|------|
| **深度** | 生产级——按 M0-M7 里程碑组织，每个里程碑有明确的 UI 状态清单 |
| **广度** | 全部页面和模块（编辑器、分支图、条件编辑器、大纲、导出、设置、模板、欢迎页） |
| **交互性** | 关键路径可交互原型（核心闭环 + 模式切换 + 入门任务） |
| **时间意图** | 逐步实现——M0 先搭建布局骨架，M1-M7 逐步填充功能模块 |

## 5. Layout Strategy

### 5.1 总布局

```
┌─────────────────────────────────────────────────┐
│  MenuBar (原生, macOS 置顶 / Win 内嵌)           │
├─────────────────────────────────────────────────┤
│  Toolbar (单行, 12 个按钮, 分组 + 分隔符)        │
├──────────┬───────────────────────┬──────────────┤
│          │                      │              │
│  Outline │   Monaco Editor      │ Branch Graph │
│  (可折叠) │   (弹性, 最小 60%)    │ (迷你地图 /   │
│  200px   │                      │  并排 40%)   │
│          │                      │              │
├──────────┴───────────────────────┴──────────────┤
│  ProblemPanel (底部滑出, 默认隐藏, Ctrl+Shift+M)  │
├─────────────────────────────────────────────────┤
│  StatusBar (单行: 保存状态 | 节点/选项数 | 错误计数 | 行:列 | 缩放 | 语言) │
└─────────────────────────────────────────────────┘
```

### 5.2 分支图双模式

| 模式 | 触发 | 形态 | 用途 |
|------|------|------|------|
| **迷你地图**（默认） | 启动即此模式 | 右下角 200×150px 窗口，hover 放大至 320×240 | 写作时保持结构感知，不占空间 |
| **并排均衡** | 点击迷你地图展开图标 / `Ctrl+Shift+G` | 右侧 40% 全高面板，完整交互 | 梳理分支逻辑、连线编辑 |

**模式切换动画**：300ms `ease-out`，迷你地图通过 `transform: scale()` + `transform-origin: bottom right` 展开为全高面板。反向收起同理。

### 5.3 渐进式披露规则

| 功能 | 默认可见？ | 首次触发 |
|------|:---:|------|
| 编辑器 | ✅ 始终 | — |
| 大纲视图 | ✅ 默认可见，可折叠 | — |
| 分支图（迷你地图） | ✅ 默认可见 | — |
| 分支图（并排） | ❌ | 点击迷你地图展开图标 |
| 条件编辑器 | ❌ | 点击选项行 `[🔧条件]` 图标 |
| 问题面板 | ❌ | `Ctrl+Shift+M` 或点击状态栏错误计数 |
| 补全候选下拉 | ❌ | `Ctrl+Space` |
| 导出对话框 | ❌ | `Ctrl+E` 或菜单 |
| 设置面板 | ❌ | 菜单 → 设置 |

### 5.4 M0 布局骨架（最小可见）

M0 不需要完整功能——只需要：

```
┌──────────────────────────────┐
│  Electron 原生菜单            │
├──────────────────────────────┤
│  [占位 Toolbar]              │
├────────┬──────────┬─────────┤
│ 占位   │ Monaco   │ 占位    │
│ 大纲   │ 编辑器   │ 分支图  │
│        │ (基础     │         │
│        │  文本编辑)│         │
├────────┴──────────┴─────────┤
│  ✅ 已保存 | 0 节点 | 100%  │
└──────────────────────────────┘
```

三个占位面板只是灰盒子 + 标签文字。Monaco 编辑器可以输入纯文本（无语法高亮）。布局可拖拽调整比例。这就是 M0 的全部 UI——验证 Electron 窗口 + Vite 构建 + 三栏布局 + 基础编辑器挂载。

## 6. Key States

### 6.1 全局状态

| 状态 | 触发条件 | 视觉呈现 | 用户感知 |
|------|---------|---------|---------|
| **欢迎** | 首次启动 / 无最近文件 | 中央欢迎对话框：Logo + 4 个模板卡片 + "空白文件"按钮 + "打开已有文件"链接 | "我可以从这里开始" |
| **就绪** | 已打开 `.mdstory` | 三栏布局，编辑器显示内容 | "一切正常，继续工作" |
| **保存中** | 停止输入 500ms | 状态栏 ⏳ 图标 | "正在保存" |
| **已保存** | 写入磁盘完成 | 状态栏 ✅ + 文件名 | "安全了" |
| **未保存** | 有修改未写入 | 状态栏 ● + 文件名旁 * | "记得保存" |
| **外部修改** | 文件被外部程序修改 | 顶部横幅："文件已被外部修改 [刷新] [另存为]" | "需要处理冲突" |
| **加载中** | 打开大文件 | 编辑器区域骨架屏 + 状态栏进度 | "马上就好" |

## 7. Interaction Model

### 7.1 入门流程

```
首次启动
  → 欢迎对话框（模板选择）
  → 用户选择模板或空白文件
  → 编辑器加载内容
  → 右下角浮现任务面板（半透明, 不遮挡内容）
  → 任务 1: "创建你的第一个节点" — 提示文字在编辑器光标旁
  → 用户完成 → 微动画 + 任务 1 ✓
  → 任务 2 浮现…
  → 全部 4 个任务完成 → 任务面板折叠为小旗图标 → 用户"毕业"
```

### 7.2 日常编辑流

```
用户在编辑器中输入
  → 500ms debounce
  → 解析器运行（后台, 不阻塞输入）
  → AST 更新
  → 同时触发:
      ├→ 分支图增量更新（新增/修改的节点 diff）
      ├→ 语法检查器 → 诊断列表 → Monaco markers 更新
      ├→ 大纲视图节点树更新
      ├→ 补全引擎异步计算
      └→ 自动保存写入磁盘
```

### 7.3 图形 → 文本同步

```
用户在分支图中拖拽连线端点
  → useGraphStore.updateEdgeTarget()
  → AST 中对应 option.targetFullId 修改
  → serializePlotFlowData() 生成新 Markdown 文本
  → Monaco editor.executeEdits() 替换文本（保留撤销栈）
  → 编辑器显示新的 -> 节点：目标
```

### 7.4 条件编辑器双向同步

```
面板 → 文本:
  用户在面板修改条件 → 实时预览更新
  → 点击"应用" → 生成条件表达式字符串
  → 更新 AST 中 option.conditions
  → serializeOptionText() → Monaco 文本替换

文本 → 面板:
  用户在编辑器中手动修改条件: 行
  → 500ms debounce → Parser 重新解析
  → AST 更新 → 面板读取新的 conditions
  → 面板刷新 ConditionRow 组件
```

### 7.5 模式切换

```
迷你地图 → 并排:
  点击展开图标 或 Ctrl+Shift+G
  → 迷你地图 scale(1) + origin bottom-right 动画 (300ms ease-out)
  → 展开为 40% 宽全高面板
  → 加载完整 React Flow 交互（Controls + Minimap 显示）

并排 → 迷你地图:
  点击折叠图标 或 Esc（焦点在分支图时）
  → 全高面板 scale(0.25) + origin bottom-right 动画 (300ms ease-out)
  → 收缩为 200×150 右下角窗口
  → 仅显示节点概览，交互禁用（只读）
```

### 7.6 键盘快捷键（核心 10 个）

| 快捷键 | 功能 | 为什么这 10 个最重要 |
|--------|------|-------------------|
| `Ctrl+S` | 强制保存 | 肌肉记忆，所有编辑器通用 |
| `Ctrl+Z` / `Ctrl+Y` | 撤销/重做 | 编辑基础 |
| `Ctrl+F` | 搜索节点/文本 | 大型项目导航 |
| `Tab` | 接受幽灵补全 | 补全核心交互 |
| `Esc` | 关闭弹出面板/忽略补全 | 退出任何模态 |
| `Ctrl+Shift+G` | 切换分支图模式 | 叙事设计师高频操作 |
| `Ctrl+Shift+C` | 打开条件编辑器 | 条件编辑入口 |
| `Ctrl+E` | 导出对话框 | 闭环终点 |
| `Ctrl+Shift+T` | 切换主题 | 亮色 ↔ 暗色 |
| `Ctrl+Shift+M` | 问题面板 | 错误诊断入口 |

## 9. M0-M7 里程碑 UI 规划
